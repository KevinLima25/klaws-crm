# MODO AUTÔNOMO

Enquanto o usuário estiver ausente:
- executar o sprint atual;
- tomar apenas decisões técnicas reversíveis;
- preservar regras de negócio;
- testar;
- documentar;
- gerar commit;
- iniciar o próximo sprint autorizado.

Criar ou atualizar `.ai/PENDENCIAS_EXECUCAO_AUTONOMA.md` com:

- Decisões de negócio
- Decisões arquiteturais
- SQL manual
- Configurações externas
- Credenciais ou acessos
- Ações de produção
- Itens para revisão

Não repetir perguntas equivalentes.

Segurança:
- nunca expor segredos;
- nunca criar credenciais falsas;
- nunca apagar dados;
- nunca fazer force push;
- nunca reescrever histórico Git;
- fazer secret scan antes da entrega final.

Um commit por sprint.
